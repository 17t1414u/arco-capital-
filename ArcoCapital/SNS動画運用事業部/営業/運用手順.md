# 制作代行 営業運用手順 v1

**対象**: SNS動画運用事業部 営業フロー (DM 初回応答 → 見積もり → 受注 → 納品 → 着金)
**Phase 1 ゲート条件**: `DM_SLO` (全社共通 #1) — Week1 達成率 100%
**SLO**: 初回 DM 返信 24 時間以内 (土日祝は翌営業日朝まで)
**モード**: B (Semi-live) — 外部 DM 送信は 15 分以内オーナー承認ゲート経由

## 1. 初回 DM 受信時の動作 (SLO 24 時間厳守)

### 1.1 受信後 10 分以内 (現場で)

1. DM の全文を読み、以下を判定:
   - 対象ペルソナ (① 治療院 / ② 美容室 / ③ 士業) のいずれかに近いか
   - 明らかな対象外 (大企業 / 顔出し希望 / 情報密度高い動画) か
   - 問い合わせ内容が空 / 意味不明 / スパム (「フォロバお願いします」等) か

2. **判定結果に応じた初動**:

| 判定 | 初動 (10 分以内) |
|------|----------------|
| 対象ペルソナに合致 | 定型返信 + ヒアリングフォームを送る (下記 1.2) |
| 対象外 (善意) | 定型のお断り文 + 本 LP の「対象外」セクション URL を返す |
| スパム | 返信せず、X のブロック機能を使う |

### 1.2 定型返信 (対象ペルソナ合致時)

以下をコピペして DM 返信する:

```
ご相談ありがとうございます、<お名前> さま。

縦型動画の制作代行を担当している <自分の名前> です。
ご依頼内容を具体化するため、以下のヒアリングフォームにご記入いただけますでしょうか。10〜15 分で回答いただけます。

[ヒアリングフォーム全文を貼付 — ArcoCapital/SNS動画運用事業部/営業/ヒアリングフォーム.md]

回答いただいてから 24 時間以内 (土日祝は翌営業日朝まで) に絵コンテ草案を作成してお送りします。

ご不明な点があれば、その旨もこの DM 内でお知らせください。
```

### 1.3 問い合わせログへの記入 (DM 返信と同じ日のうち)

`問い合わせログ.md` の「1. アクティブ案件」表に 1 行追加する:

```markdown
| N | 2026-04-YY | @xxxxxxxxx (匿名ID) | 治療院 / 美容室 / 士業 / その他 | YYYY-MM-DD | ¥15,000 / ¥30,000 / 未定 | 相談中 | (空白 — フォーム回答後に slug を付与) | 初回 DM 受信 HH:MM / 返信 HH:MM |
```

- `匿名ID`: X の `@ユーザー名` をそのまま記載 (DM 相手の同意なく実名は書かない)
- `案件 slug`: ヒアリングフォーム回答受領後に `YYYY-MM-DD_<業種>-<連番>` で命名し追記
- 備考には **DM 受信時刻と返信時刻の両方** を JST 表記で (SLO 計測のため)

### 1.4 その場で返信できない場合 (外出中等)

- **6 時間以内の簡易返信**: 「ご相談ありがとうございます。明日 朝 ●時までに正式な返信をお送りします」だけでも送る (SLO の「24 時間」は完全回答ではなく「最初の一声」で計測可)
- **24 時間以内に正式返信**: 必ず 1.2 のヒアリングフォーム付き返信を送る
- **24 時間超過が発生した場合**: `operations/incident_log.jsonl` に SLO 違反として記録 + COO にエスカレーション

## 2. ヒアリングフォーム回答受領時

### 2.1 内容確認 (1 時間以内)

- [ ] 必須欄 (業種 / 希望納期 / 予算 / 3 キーワード / NG カット) がすべて埋まっているか
- [ ] 予算 (¥15,000 / ¥30,000) と `CLAUDE.md` の価格固定方針の整合 (¥10,000 未満は受注しない)
- [ ] 希望納期 7 営業日が間に合うか (受注が連続している週は翌週繰越を事前説明)
- [ ] NG カットに医療行為・治療行為・特定店舗外観が含まれていないか

### 2.2 絵コンテ起票 (Day 2-3)

回答完了後、以下を起票:

1. `ArcoCapital/SNS動画運用事業部/企画/<YYYY-MM-DD>_<slug>/` ディレクトリ作成
2. `project.json` (cinematic-everyday-01 と同スキーマで shots: 3、各 10s)
3. `storyboard.md` (依頼主の 3 キーワード・情感・NG カットを反映した S1-S3 文字コンテ)
4. 完成した `storyboard.md` を DM で送信し OK / 修正指示を待つ

### 2.3 ログ更新

`問い合わせログ.md`:
- 現ステータスを「相談中」→「絵コンテ中」に更新
- 案件 slug をカラムに記入
- 備考に「ヒアリング回答受領 YYYY-MM-DD HH:MM」を追記

## 3. 案件進行中のステータス遷移

`問い合わせログ.md` のステータス定義:

| 遷移元 | イベント | 遷移先 |
|--------|---------|--------|
| 相談中 | ヒアリング回答受領 | 絵コンテ中 |
| 絵コンテ中 | 絵コンテ送信 | 承認待ち |
| 承認待ち | OK 返信 | 生成中 |
| 承認待ち | 修正指示 | 絵コンテ中 (無制限ループ可) |
| 承認待ち | 辞退返信 | キャンセル (着金ゼロで完了) |
| 生成中 | 3 セグ全 L3 パス | 編集中 |
| 編集中 | ffmpeg 結合 + BGM | 納品済 |
| 納品済 | 着金連絡受領 | 入金待ち |
| 入金待ち | 着金確認 | 完了 |

ステータス更新時には `問い合わせログ.md` の該当行を編集し、備考に遷移時刻を追記する。

## 4. 週次レビュー (土曜 19:00 JST)

### 4.1 問い合わせログの振り返り

毎週土曜、以下を `問い合わせログ.md` の「4. 月次サマリ」に追記:

- [ ] 今週の問い合わせ件数 (アクティブ + キャンセル + 完了)
- [ ] 初回応答 SLO 遵守率 (計算式: (24h 以内返信件数) / (全問い合わせ件数) × 100)
- [ ] 今週の売上 (完了案件の金額合計)
- [ ] 今週のキャンセル理由トップ 3

### 4.2 BoardMeetingCrew への提出

- 上記の数値を取締役会の議事録 (`ArcoCapital/経営陣/取締役会議事録/YYYY-MM-DD_*.md`) に自動添付される形に整える (現状は手動コピペ)
- COO がレビュー → 来週の改善アクションを決議

### 4.3 撤退基準チェック (5/18 が 1 ヶ月目)

月次 ¥100,000 目標に対する進捗を確認:
- 50% 未達 (¥50,000 未満): 事業縮小再設計
- 80% 未達 (¥80,000 未満): 改善アクション強化 (Mode B 延長)
- 100% 達成 (¥100,000 以上): Mode C 昇格判定

## 5. 失注 / キャンセル処理

### 5.1 失注ケース (見積もり提出後に辞退)

`問い合わせログ.md` の「3. キャンセル / 失注」表に記入:

| 項目 | 記入内容 |
|------|---------|
| 受注日 | 初回 DM 受信日 |
| 失注日 | 辞退 DM 受信日 |
| 業種 | ペルソナ分類 |
| 予算 | 想定予算帯 |
| 失注理由 | (下記カテゴリから選択) |
| 学び | 1 行で簡潔に (次回改善のため) |

失注理由カテゴリ: 予算不一致 / 納期不一致 / 顔出し希望 / 他社決定 / プロジェクト中止 / レスポンス途絶 / その他

### 5.2 キャンセル (絵コンテ段階での辞退)

- 着金ゼロでクローズ → 問い合わせログの「アクティブ案件」から削除し、「キャンセル / 失注」に移動
- 備考に「絵コンテ段階キャンセル — 原因: 方向性不一致 / 予算見直し / etc.」を記入

### 5.3 キャンセル (本編生成後の辞退)

- 生成実費 ¥5,000 を請求 (LP「お支払い・キャンセル条件」に基づく)
- 着金確認後、完了扱い (売上に計上)
- 生成済の mp4 は依頼主に納品しない (権利関係の混乱を避けるため、本編生成実費は「制作過程費」として扱う)

## 6. 想定されないケース (エスカレーション)

以下のケースに遭遇した場合は、いったん返信を保留し、オーナー (自分) に 15 分以内にメモを残す:

- 法令違反を疑わせる業種 (未認可医療行為・金融商品勧誘 etc.)
- 未成年を被写体にしたい依頼
- 競合他社からの偵察的問い合わせ (明らかに制作能力を探るだけの内容)
- 誹謗中傷を含む動画制作依頼

これらは **受注しない** 方針。お断り返信の雛形を作って送る。

前月の完了案件を `実績/[YYYY-MM]/` にポストモーテムとしてアーカイブ:

1. ディレクトリ作成: `ArcoCapital/SNS動画運用事業部/実績/2026-05/`

3. `実績/2026-05/月次レポート.md` を作成し、月次売上・件数・学びを記録
4. `問い合わせログ.md` の「2. 完了案件」表の「ポストモーテム」欄に相対パスを記入
