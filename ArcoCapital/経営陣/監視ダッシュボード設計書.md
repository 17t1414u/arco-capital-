> **根拠**: `.octogent/decision.md` § 4 合意条件 — CTO/COO 拘束条件「監視ダッシュボード(エラー率/SLO/コスト)稼働」
> **担当**: CTO (実装) / COO (運用)
> **ステータス**: v0.1 (Phase 0 暫定・4/25 ゲート前に稼働必須)

---

## 1. 要件 (MUST)

Phase 1 ゲート (2026-04-25) までに以下 **3指標** を可視化する:

| カテゴリ | 指標 | 取得元 | 閾値 (Amber/Red) |
|---------|------|--------|------------------|
| **エラー率** | Director エージェント呼び出しのエラー率 (日次) | CrewAI ログ / `tasks/**/outputs/*` | Amber ≥5% / Red ≥10% |
| **SLO** | DM 返信までの経過時間 (中央値・P95) | X DM ログ (手動集計可) | Amber ≥20h / Red ≥24h |
| **コスト** | Anthropic API + Renoise クレジットの消費推移 | `operations/budget_log.jsonl` | Amber ≥予算の70% / Red ≥90% |

---

## 2. 実装方針 (Phase 0 暫定)

**"最低限動く" 優先**。凝った BI ツールは入れない。

### 2.1 データレイヤ
- `operations/budget_log.jsonl` — Anthropic API / Renoise 消費ログ (BudgetTracker が追記)
- `operations/incident_log.jsonl` — Kill-switch 発火・インシデント履歴
- `ArcoCapital/**/outputs/*.md` — 各 Director の実行出力 (エラー含む)
- `X_DM_log.csv` (手動) — オーナーが日次で DM 応答時間を記録

### 2.2 可視化レイヤ
**v0.1 (本 PR)**: Markdown レポート自動生成

```bash
python -m operations.dashboard_report --date 2026-04-22
```

生成先: `ArcoCapital/経営陣/監視レポート/YYYY-MM-DD.md`

フォーマット:
```markdown
# 監視レポート 2026-04-22

## エラー率 (日次)
- Director 呼び出し総数: 12
- エラー: 1 (8.3%) — Amber ⚠

## SLO (DM返信)
- 対象 DM: 3
- 中央値: 4.5h
- P95: 18h — Green ✅

## コスト
- Anthropic API: ¥870 / ¥1,500 (58%) — Green ✅
- Renoise: ¥3,200 / ¥10,000 (32%) — Green ✅
```

**v0.2 (Phase 1 以降)**: Streamlit or Obsidian Dataview プラグインでリアルタイム化検討

## 3. 運用フロー

```
  [日次 23:55 (JST)]
         │
         ▼
  cron / タスクスケジューラで
  python -m operations.dashboard_report --date today を実行
         │
         ▼
  ArcoCapital/経営陣/監視レポート/YYYY-MM-DD.md 生成
         │
         ▼
  オーナーが翌朝確認 (5分)
         │
         ├─ Red 指標あり → Kill-switch 発火 or 該当 Director を Mode A 降格
         ├─ Amber 指標あり → 次回取締役会で議題化
         └─ All Green   → 何もしない
```

---

## 4. 閾値超過時の連動

| 指標 | Red 到達時のアクション |
|------|----------------------|
| エラー率 ≥10% | 該当 Director を Mode A へ自動降格 (`guardrails.yaml` 書換) |
| DM-SLO P95 ≥24h | COO にメンション、手動で Director に「応答優先度UP」指示 |
| コスト ≥90% | Amber で通知、100% で Kill-switch 自動発火 (既実装) |
