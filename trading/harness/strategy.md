## PART 1: スクリーニングルール（全500銘柄に適用）
## ═══════════════════════════════════════

### フェーズ1 — 流動性フィルター（必須、全条件）
```
条件1: 日次平均出来高 ≥ 100万株（非流動銘柄を除外）
条件2: 株価 ≥ $10（ペニー株・低位株を除外）
条件3: 通常取引時間内のみ（プレ/アフターマーケット取引なし）
```
→ 約400銘柄に絞り込み

### フェーズ2 — テクニカルスクリーニング（3条件中2以上を満たす）
```
条件A: RSI(14) < 38（過売り圏 — 反発候補）
条件B: 価格 > SMA(50)（中期上昇トレンド内）
条件C: MACDヒストグラム > 0 または直近で正転換（モメンタム）
```
→ 上位20〜30銘柄に絞り込み

### フェーズ3 — ランキング（スコア順に上位10銘柄を選出）
```
スコア計算:
  RSIスコア: (38 - RSI) / 38 × 40点  ← RSIが低いほど高スコア
  ボリュームスコア: 当日出来高/20日平均出来高 × 30点（上限30点）
  モメンタムスコア: MACDヒストグラムの正転換日数 × 5点（上限30点）
  合計: 最大100点
```
→ **候補銘柄Top10**として深層分析へ

## PART 2: 投資ルール（エントリー/エグジット）
## ═══════════════════════════════════════

### エントリー条件（BUY）— 全条件を満たすこと

#### 必須条件（ハードルール）
```
[必須1] スクリーニングTop10に入っていること
[必須2] ファンダメンタルズ判定が BULLISH または NEUTRAL（BEARISHは禁止）
[必須3] ニュース判定が NEGATIVE_HIGH ではないこと（重大悪材料なし）
[必須4] センチメント判定が EXTREME_GREED ではないこと（過熱相場に乗らない）
[必須5] RiskManagerが APPROVED または REDUCE_SIZE であること
[必須6] 同一銘柄の直前取引から24時間以上経過していること
```

#### 推奨条件（Bull/Bear議論で評価）
```
[推奨1] Bull論拠の質がBear論拠を明確に上回る
[推奨2] RSI(14) < 35（より強いシグナル）
[推奨3] ボリュームが20日平均の1.5倍以上（出来高増加で確信度UP）
[推奨4] SMA(20)とSMA(50)がゴールデンクロス直後
```

### エグジット条件（SELL/HOLD→SELL）

#### 利確（Profit Taking）
```
目標1: +10%（固定利確）
目標2: RSI(14) > 72 かつ MACDデッドクロス（過熱シグナル）
目標3: トレーリングストップ: 最高値から-7%下落（利益を守る）
```

#### 損切り（Stop Loss）— 絶対遵守
```
ストップロス: エントリー価格から -5%（交渉不可）
例外なし: いかなる状況でも損切りラインを超えたら即決済
```

#### 強制終了トリガー
```
ポートフォリオ全体のドローダウンが -15% に達した場合:
  → すべてのポジションを見直し
  → 新規エントリーを一時停止
  → 人間による状況確認が必要
```

## PART 3: 資金管理ルール（Money Management）
## ═══════════════════════════════════════

### 3-1. ポジションサイジング — ケリー基準（半ケリー）

```
ケリー基準式:
  f* = (p × b - q) / b

  p = 勝率（過去取引から計算、初期値: 0.55）
  q = 敗率（= 1 - p）
  b = 平均利益 / 平均損失（初期値: 2.0 = +10% / -5%）

例（初期値）:
  f* = (0.55 × 2.0 - 0.45) / 2.0 = 0.325 (32.5%)

安全のため半ケリーを使用:
  実用ポジションサイズ = f* × 0.5 = 16.25% → 上限10%でキャップ
```

| パラメータ | 値 |
|-----------|-----|
| 最小ポジションサイズ | 総資産の2% |
| 最大ポジションサイズ | 総資産の10% |
| ケリー係数 | 0.5（半ケリー） |
| 初期勝率（実績なし時） | 55% |
| 初期利益率/損失率比 | 2.0（+10% / -5%） |

### 3-2. ポートフォリオ構成ルール

```
最大同時保有銘柄数: 10銘柄
最大総エクスポージャー: 総資産の80%（20%は常時キャッシュ確保）
1セクター最大集中度: 30%（特定セクターへの過剰集中防止）
相関フィルター: 相関係数0.85以上の銘柄は同時保有しない
```

### 3-3. リスク予算

| リスク指標 | 警告水準 | 停止水準 |
|----------|---------|---------|
| ポートフォリオドローダウン | -10% | -15% |
| 1日最大損失 | -3% | -5% |
| 週次最大損失 | -5% | -8% |
| 1日最大取引回数 | — | 5回 |

### 3-4. ケリー係数の動的更新

```
取引が50件以上蓄積した時点でOptimizerが自動計算:
  p = 実際の勝率
  b = 実際の平均利益 / 平均損失

毎月のオプティマイザーループで strategy.md の
初期勝率・利益率/損失率比を実績値に更新する。
```
