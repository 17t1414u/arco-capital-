### v2.0 → v2.1 の変更点
- **逆張り経路 (Path A)** の RSI BUY 閾値を 35 → **45** に緩和（候補数増加）
- **順張り経路 (Path B)** を新規追加（健全な上昇圏でのモメンタム買い）
- フェーズ2は 5条件に拡張、Path A/B いずれか満たせば候補入り
- フェーズ3 スコアを経路別に再設計

### v2.1 → v2.2 の変更点（売却条件の緩和）
- **損切り**: -5% → **-7%** （ノイズ吸収、過剰早期売却を防ぐ）
- **強制利確 TP+** (Layer A 即SELL): +20% → **+30%** （ウィナーを伸ばす）
- **利確目標1** (Layer B 判断): +10% → **+12%** （LLMにトレンド継続性判断の余地）
- **過熱SELL**: RSI>72+デッドクロス → **RSI>78+デッドクロス+出来高減少** （単発で売らない）
- **トレーリングストップ**: 高値から -7% → **-10%** （一時的下落で売り急がない）
- **TRIM 部分利確**: +15% → **+20%** （より高水準で半分利確）
- 強制終了 (ポートフォリオドローダウン -15%) は維持

### v2.2 → v2.3 の変更点（トリプルパス化、機会損失の解消）
**目的**: 強気相場の真っ最中でトレードが全く走らない設計欠陥を解消。
- **Path A 拡張（押し目買い）**: RSI < 45 → **< 50**、+「直近3日下落-3%以上」を追加条件化
- **Path B 拡張（モメンタム順張り）**: RSI上限 70 → **75**、出来高条件 1.3x → **1.2x**
- **Path C 新設（過熱反転ショート）**: RSI>75 + 過剰乖離 + MACD減速 + 出来高萎縮で意図的SHORT
- **エントリー方向**: LONG-only → **LONG/SHORT 両建て**（Path C 経由のみ SHORT 許可）
- **ガードレール拡張**: PlaceOrderTool に `intentional_short` フラグ追加（Path C 経由のみ通過）
- エグジット条件 (Layer A/B) は v2.2 から変更なし

## PART 1: スクリーニングルール（全500銘柄に適用）
## ═══════════════════════════════════════

### フェーズ1 — 流動性フィルター（必須、全条件）
```
条件1: 日次平均出来高 ≥ 100万株（非流動銘柄を除外）
条件2: 株価 ≥ $10（ペニー株・低位株を除外）
条件3: 通常取引時間内のみ（プレ/アフターマーケット取引なし）
```
→ 約400銘柄に絞り込み

### フェーズ2 — テクニカルスクリーニング（3経路トリプルパス）

候補入りには **Path A（逆張り買い）/ Path B（順張り買い）/ Path C（過熱反転売り）** いずれかを満たすこと。

#### 🅰️ Path A: 逆張り買い（押し目買い・反発狙い）— LONG
**A1 + A2 + (A3a or A3b) を満たす:**
```
条件A1: RSI(14) < 50（過売り〜中立、調整入り銘柄を捕捉）
条件A2: 価格 > SMA(50)（中期上昇トレンドが続いている = 健全な押し目）
条件A3a: MACDヒストグラム > 0 または直近5日で正転換（モメンタム回復）
条件A3b: 直近3営業日で-3%以上下落（明確な押し目検知）
```

#### 🅱️ Path B: 順張り買い（モメンタム・トレンドフォロー）— LONG
**全4条件を満たす:**
```
条件B1: RSI(14) ∈ [50, 75]（健全圏〜強気継続中、上限拡大）
条件B2: SMA(20) > SMA(50)（パーフェクトオーダー = 短期>中期上昇）
条件B3: 価格 > SMA(20)（直近のサポートを保持）
条件B4: 出来高 ≥ 20日平均の1.2倍（資金流入の裏付け、緩和）
```

#### 🅲️ Path C: 過熱反転売り（意図的ショート）— **SHORT**
**全4条件を満たす（≪Path C 銘柄のみ意図的ショート建てを許可≫）:**
```
条件C1: RSI(14) > 75（極度の過熱）
条件C2: 価格 vs SMA(20) 乖離 ≥ +10%（過剰乖離 = 巻き戻しリスク）
条件C3: MACDヒストグラムが減速中（前日値より小さい = ピーク打ちサイン）
条件C4: 出来高 ≤ 20日平均の0.9倍（買い疲れ・息切れシグナル）
```
**重要**: Path C のみ `intentional_short=True` フラグで Alpaca に SELL 注文（=ショート建て）を許可。
それ以外の保有なし SELL は PlaceOrderTool ガードで物理的にブロック。

→ Path A / B / C のいずれかを満たせば次フェーズへ

### フェーズ3 — スコアランキング（経路別、上位10銘柄を選出）

#### Path A スコア（最大100点）
```
RSIスコア:    (50 - RSI) / 50 × 35点  ← RSIが低いほど高スコア
出来高スコア: min(当日/20日平均, 3.0) / 3.0 × 25点
モメンタム:   MACD正転換=20点 / MACDヒスト>0=10点 / なし=0
押し目強度:   直近3日下落幅 ×絶対値 ×4 (最大20点)
合計: 最大100点
```

#### Path B スコア（最大100点）
```
トレンドスコア: 価格 vs SMA50 乖離率 × 3点（上限35、+12%付近で満点）
出来高スコア:   min(当日/20日平均, 3.0) / 3.0 × 25点
モメンタム:     MACDヒスト符号化 25点（>0で満点、増加中ならボーナス+5）
RSI健全度:      RSI 60付近で満点(15点)、極端値で減点
合計: 最大100点
```

#### Path C スコア（最大100点）— ショート期待値
```
過熱度スコア:   (RSI - 75) / 25 × 30点（RSIが高いほど高スコア、最大30）
乖離スコア:     (価格/SMA20 - 1.10) × 100 × 5点（+10%超過分、最大25点）
減速スコア:     MACDヒスト減速幅 × 25点
出来高萎縮:     (1 - 出来高比) × 20点（萎縮幅大きいほど高、最大20）
合計: 最大100点
```

→ **3経路の合算結果からスコア上位Top10**として深層分析へ
   （LONGとSHORTを混合してランキング、エントリー方向は経路で自動決定）

## PART 2: 投資ルール（エントリー/エグジット）
## ═══════════════════════════════════════

### エントリー条件（BUY）— 全条件を満たすこと

#### 必須条件（ハードルール）
```
[必須1] スクリーニングTop10に入っていること
[必須2] ファンダメンタルズ判定が BULLISH または NEUTRAL（BEARISHは禁止）
[必須3] ニュース判定が NEGATIVE_HIGH ではないこと（重大悪材料なし）
[必須4] センチメント判定が EXTREME_GREED ではないこと（過熱相場に乗らない）
[必須5] RiskManagerが APPROVED または REDUCE_SIZE であること
[必須6] 同一銘柄の直前取引から24時間以上経過していること
```

#### 推奨条件（Bull/Bear議論で評価）
```
[推奨1] Bull論拠の質がBear論拠を明確に上回る
[推奨2] Path A 経路の場合: RSI(14) < 35（より強い反発シグナル）
        Path B 経路の場合: RSI(14) ∈ [55, 65]（最も健全なモメンタム圏）
[推奨3] ボリュームが20日平均の1.5倍以上（出来高増加で確信度UP）
[推奨4] SMA(20)とSMA(50)がゴールデンクロス直後（中期トレンド転換）
[推奨5] 直近5日でMACD正転換（モメンタム回復のタイミング）
```

### エグジット条件（SELL/HOLD→SELL） — v2.2 バランス型

エグジット判断は2層構造で実施:
- **Layer A (Hard Rules)**: 決定論的、LLM不要、即SELL
- **Layer B (LLM Judge)**: ニュアンス判断（テクニカル + ニュース + センチメント）

#### Layer A（即SELL — 機械的判定）
```
[A-SL] 損切り: エントリー価格から -7% に到達 → 即全量SELL（絶対遵守）
[A-TP+] 強制利確: +30% 以上の含み益 → 即全量SELL（過剰利益確定）
```

#### Layer B（LLM 判断ベース）
含み益 +12% 〜 +30% の範囲、または以下のシグナルでLLMが評価:
```
[B-TP1] 利確目標1: +12% 以上 → トレンド継続性をLLMが評価
        - 継続中(MACD強気維持)なら HOLD
        - 鈍化なら SELL or TRIM(半分)

[B-TRIM] 部分利確: +20% 以上 → デフォルト TRIM(50%売却)、残りはトレーリング監視

[B-OVERHEAT] 過熱SELL: 以下を全て満たす場合 SELL
        - RSI(14) > 78
        - MACDデッドクロス（直近5日以内）
        - 出来高が20日平均を下回る（息切れシグナル）

[B-TRAIL] トレーリングストップ: 直近30日高値から -10% 下落
        - 大きな反転とLLMが判断すれば SELL
        - 一時的調整と判断すれば HOLD
```

#### 強制終了トリガー
```
ポートフォリオ全体のドローダウンが -15% に達した場合:
  → すべてのポジションを見直し
  → 新規エントリーを一時停止
  → 人間による状況確認が必要
```

## PART 3: 資金管理ルール（Money Management）
## ═══════════════════════════════════════

### 3-1. ポジションサイジング — ケリー基準（半ケリー）

```
ケリー基準式:
  f* = (p × b - q) / b

  p = 勝率（過去取引から計算、初期値: 0.55）
  q = 敗率（= 1 - p）
  b = 平均利益 / 平均損失（初期値: 2.0 = +10% / -5%）

例（初期値）:
  f* = (0.55 × 2.0 - 0.45) / 2.0 = 0.325 (32.5%)

安全のため半ケリーを使用:
  実用ポジションサイズ = f* × 0.5 = 16.25% → 上限10%でキャップ
```

| パラメータ | 値 |
|-----------|-----|
| 最小ポジションサイズ | 総資産の2% |
| 最大ポジションサイズ | 総資産の10% |
| ケリー係数 | 0.5（半ケリー） |
| 初期勝率（実績なし時） | 55% |
| 初期利益率/損失率比 | 2.0（+10% / -5%） |

### 3-2. ポートフォリオ構成ルール

```
最大同時保有銘柄数: 10銘柄
最大総エクスポージャー: 総資産の80%（20%は常時キャッシュ確保）
1セクター最大集中度: 30%（特定セクターへの過剰集中防止）
相関フィルター: 相関係数0.85以上の銘柄は同時保有しない
```

### 3-3. リスク予算

| リスク指標 | 警告水準 | 停止水準 |
|----------|---------|---------|
| ポートフォリオドローダウン | -10% | -15% |
| 1日最大損失 | -3% | -5% |
| 週次最大損失 | -5% | -8% |
| 1日最大取引回数 | — | 5回 |

### 3-4. ケリー係数の動的更新

```
取引が50件以上蓄積した時点でOptimizerが自動計算:
  p = 実際の勝率
  b = 実際の平均利益 / 平均損失

毎月のオプティマイザーループで strategy.md の
初期勝率・利益率/損失率比を実績値に更新する。
```
