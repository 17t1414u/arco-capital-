## なぜ「外部脳」が必要か ― 3人の失敗ログ

### ① 20代OL・鈴木さん(仮)のケース
副業で note を始めて3ヶ月。X で拾った投資記事を5本メモしたが、いざ自分の note を書こうとした時に **「どこに保存したか思い出せない」** 。ブラウザのブックマーク→スプレッドシート→スマホのメモアプリ、と分散していた。結果、同じ記事を2度調べ直し、週末2時間を消費。

**解決**: Vault の `raw/` に全部放り込んで `/wiki-ingest` すれば、3秒で全文検索できる。

### ② 30代会社員・田中さん(仮)のケース
月4本の note を半年続けて、合計24本の資産が貯まった。しかし新しい記事を書く時、過去記事と矛盾する主張をしていないか **毎回チェックする時間がない** 。結果、読者から「3ヶ月前と意見が変わってますよ」と指摘されて信頼を落とした。

**解決**: `/wiki-lint` を週1で回せば、過去発言との矛盾を自動検出できる。

### ③ 40代女性・佐藤さん(仮)のケース
子どもが寝た後の1時間で、週2本の note 更新が目標。だが毎回 **「前回どこまで書いたっけ?」** で20分、**「今日は何を調べたっけ?」** で10分。実質30分しか書けず、月8本予定が4本しか出ない。

**解決**: 日次振り返りテンプレに3行書くだけで、翌日の1行目がすぐ見つかる。

### この3つの痛みと、このキットの3テンプレの対応

| 痛み (上の失敗ログ) | 解消するテンプレ | 効き方 |
|-------------------|----------------|-------|
| ① 情報が分散して再調査する | 記事執筆テンプレ (参照ソース欄) | 書き始める前に参照ソースを3本固定 |
| ② 過去の主張と矛盾する | 研究ログテンプレ + `/wiki-lint` | 週1で矛盾・ギャップを洗い出す |
| ③ 前回の続きを思い出せない | 日次振り返りテンプレ (明日1行目) | 昨日の最後の1文が今日の1行目 |

次章で3テンプレの中身をそれぞれ見ていきます。

## キット内容 ― 3つのテンプレートと何が違うのか

### テンプレ1: 記事執筆テンプレート (`templates/v1/article.md`)

```
# {タイトル}
## リード(結論1文)
## 想定読者(ペルソナ3名・年代と年収)
## 本文
## 参照ソース(Vault 内リンク3本以上)
## 次に書くための残弾(Wiki タグ3つ)
```

**何が違うか**: 書き始める前に **参照ソース3本と残弾3タグ** を埋める欄がある。これだけで「書きながら調べ直し」がゼロになる。

**使った人の実測(筆者)**: 筆者の直近5記事 (2,000〜3,000字帯・副業レビュー系) の執筆時間平均が **約90分 → 約52分** に短縮。リサーチを書き始める前に3本で固定しているため、本文中に再調査が挟まらないことが主な要因です。計測条件が揃っても他の人が同じ比率になる保証はなく、まずは1週間回して、自分の所要時間を測ってから判断してください。

### テンプレ2: 研究ログテンプレート (`templates/v1/research.md`)

```
# {テーマ}
## ソース一覧(URL + 1行要約 + 取得日)
## 仮説3つ
## 反証3つ(必須)
## Synthesis(統合所感・引用元ページ番号まで)
## 派生テーマ候補(将来の記事タネ)
```

**何が違うか**: **反証3つ欄が必須** 。調べている最中に「この主張に不利な事実・反例」を3本メモする手が止まる作りなので、都合のいい引用だけで記事を組む流れを自分で遮断できます。読者から「反論材料を知っていて書いたのか」と問われた時に、自分の調査メモで答えられる状態を作る欄です (読者の反応そのものを保証するものではありません)。

### テンプレ3: 日次振り返りテンプレート (`templates/v1/daily.md`)

```
# {YYYY-MM-DD}
## 今日書いた(記事タイトル・文字数)
## 今日読んだ(Vault raw/ に入れた本数)
## 未消化の疑問(最大3つ・翌日の1行目になる)
## 明日1行目に書くこと(1文だけ)
```

**何が違うか**: 「明日1行目に書くこと」を **1文で残す** 。翌日、エディタを開いた瞬間に書き出せるので「何を書くか考える時間」がゼロ。

## この記事のまとめ

外部脳は「書く力を上げる道具」ではなく、**「忘れない仕組み」** です。書く力は元から足りています。足りないのは、昨日の自分が書いたものを今日の自分が取り出せる経路だけです。

**やってほしいこと**
- Obsidian を公式サイトからダウンロードして、空の Vault を1つ作る
- `raw/` `wiki/` `templates/` の3フォルダを作り、本キットのテンプレ3枚を `templates/v1/` に置く
- 日次振り返りテンプレに **「明日1行目に書くこと」を1文だけ** 残す習慣をつける

**やらなければいけないこと(前提)**
- Claude Code CLI を事前にインストールし、`/wiki-ingest` が自分の Vault で動く状態にしておく
- Vault はローカル運用で構わないが、クラウド同期(iCloud / OneDrive / GitHub 等)のどれかは必ず設定し、バックアップを二重化する

**重要な考え方**
- テンプレは「埋める欄を減らす」のが目的であって、「完璧に埋める」のが目的ではない
- 参照ソースは3本で足りる。10本貼っても読者が追うのはせいぜい上から2〜3本で、増やすほど「どれが筆者の根拠か」が拡散してしまう。3本に絞って各1文ずつ要約を添える方が、説得力の密度は上がる
- 書き手の資産は「公開した記事の数」ではなく、「再利用できる形で残したソースの数」

この3枚のテンプレに沿って1週間回すと、自分がどの工程で時間を失っているかが計測できる状態になります。「書きながら調べ直す時間」が完全にゼロになる保証はありませんが、減らす対象が自分で見えるので、次に削る場所を自分で選べる状態にはなります。
